Example: Detection Fidelity applied in practice

Detection: Human lateral authentication to server (first-seen / out-of-pattern).
Hypothesis: valid user credentials being abused for lateral movement into server-class assets.
Telemetry: Windows Security 4624 (success), optional 4625 (fail), LogonType 3/10, destination classified as server.

Identity-first detection Lateral movement Windows Security Logs Server access
Initial state: SUPPRESSED Baseline: 14‚Äì30 days Promote when: first-seen + critical server Retire if: server/workstation not distinguishable
Signal strength: 4 / 5 Noise cost: 3 / 5 Response burden: 3 / 5 Telemetry reliability: 4 / 5 Maintainability: 4 / 5 Overall: 3.6 / 5 ‚Äî Promote with constraints

Operational rationale: this detection starts suppressed to build identity and host baselines. It is promoted to an alert only when it becomes actionable (first-seen access to critical servers). If the environment cannot reliably separate servers from workstations, the detection is retired to preserve SOC trust and avoid structural noise.
